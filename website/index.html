How it works

  1. Raw intent
    Buyer describes what they want in messy natural language.
  2. LLM extraction
    Gemini extracts a strict-schema JSON card. Ollama fallback if Gemini is down.
  3. Post-processing
    Deterministic rules: taxonomy match, entity extraction, flag inference.
  4. Structured Requirement Card
    Must-haves, nice-to-haves, disqualifiers, search queries, confidence.

Adversarial validator

We ship a battery of 12 hostile test cases — ambiguous, contradictory, mixed-language, hidden-constraint, and more. A downstream agent can run it to see whether NeedTranslator's current model handles the failure modes it cares about.

What this skill is for

Marketplace buyers describe things in natural language. Marketplaces index them by category, SKU, and structured attributes. Existing intent parsers either strip too much context (keyword extractors) or hallucinate constraints that were never there (LLM-only pipelines). NeedTranslator sits in the middle: LLM extraction wrapped in strict schema validation, deterministic post-processing, and a real adversarial validator so an AI agent can trust the output.

Composable. A downstream FairPrice skill can consume a Requirement Card. A TrustCheck skill can validate its constraints. NeedTranslator is designed to be the first hop in a chain of NANDA marketplace skills.
